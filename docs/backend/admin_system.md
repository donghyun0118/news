# 관리자 시스템 및 알림 (Admin System & Notifications)

## 1. 관리자 대시보드 (`src/admin`)

관리자 대시보드는 플랫폼의 제어 센터로, 관리자 권한이 있는 사용자만 접근할 수 있습니다.

### 주요 모듈

- **AdminTopics**: 토픽과 해당 기사를 생성, 편집 및 관리합니다.
- **AdminUsers**: 사용자 목록을 조회하고 사용자를 정지/차단합니다.
- **AdminInquiries**: 1:1 사용자 문의(`tn_inquiry`)를 관리합니다.
- **AdminKeywords**: 검색 키워드 및 트렌드를 관리합니다.

### 보안

- **AdminGuard**: 모든 `/api/admin/*` 라우트에 적용됩니다. JWT를 검증하고 `ADMIN` 역할을 확인합니다.

## 2. 알림 시스템 (`src/notifications`)

사용자에게 알림을 보내는 기능을 처리합니다.

### 유형

- **실시간 (Socket.IO)**: 즉각적인 인앱 업데이트용 (예: "새 메시지").
- **푸시 알림**: (계획됨/구현된 경우) 모바일 장치용.

### API 엔드포인트

- **`POST /api/admin/notifications`**:
  - **용도**: 관리자가 모든 사용자 또는 특정 그룹에게 수동으로 알림을 발송합니다.
  - **인증**: 관리자 JWT.
- **`POST /api/internal/send-notification`**:
  - **용도**: 내부 시스템(예: Python 스크립트)이 알림을 트리거합니다 (예: "새 토픽 생성됨").
  - **인증**: `x-internal-secret` 헤더. **프론트엔드 사용 불가**.

## 3. 문의 시스템 (`src/inquiry`)

사용자가 지원팀에 연락할 수 있게 합니다.

### 흐름 (Flow)

1.  사용자가 앱/웹을 통해 문의를 제출합니다.
2.  `tn_inquiry`에 저장됩니다.
3.  관리자가 대시보드에서 확인하고 답변합니다.
4.  답변이 저장되고 사용자에게 알림이 전송됩니다.
