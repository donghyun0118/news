테스트 순서:
   0. 라이브러리 설치: npm install socket.io-client

   1. 로그인: 먼저, 일반 사용자로 로그인하여 JWT 토큰을 발급받습니다.

   2. 소켓 연결 (인증 포함): 메인 페이지에 진입하면, 백엔드 소켓 서버로 연결을 시도합니다. 이때, 1번에서 받은 JWT 토큰을 auth.token 옵션에 담아 보내야 사용자 인증이 완료됩니다.

   1     import { io } from "socket.io-client";
   2 
   3     const jwtToken = "여기에_로그인해서_받은_토큰"; // 실제 토큰으로 교체
   4     const socket = io("https://news-buds.onrender.com", { // 서버 주소
   5       auth: {
   6         token: jwtToken
   7       }
   8     });

   3. 채팅방 참여: 소켓이 성공적으로 연결되면(connect 이벤트 확인 후), 메인 페이지의 채팅방 번호인 '1'을 사용하여 'join_room' 이벤트를 서버로 보냅니다.
   1     socket.on('connect', () => {
   2       console.log('소켓 연결 성공!');
   3       socket.emit('join_room', '1'); // 메인 페이지 채팅방 ID는 '1'
   4     });

   4. 이전 대화 불러오기 (HTTP API): 채팅방에 참여한 후, 이전 대화 기록을 가져오기 위해 다음 HTTP API를 호출합니다.
       * GET /api/topics/1/chat
       * 이 API는 해당 채팅방의 이전 메시지 목록을 반환하며, 이 데이터를 사용해 채팅창의 초기 화면을 구성합니다.

   5. 메시지 수신: 다른 사용자가 보낸 메시지를 실시간으로 받기 위해 'receive_message' 이벤트를 리스닝합니다.

   1     socket.on('receive_message', (data) => {
   2       // data = { message: "...", author: "..." }
   3       console.log('새 메시지 수신:', data);
   4       // 이 데이터로 채팅 UI를 업데이트합니다.
   5     });

   6. 메시지 전송: 사용자가 메시지를 입력하고 '전송' 버튼을 누르면, 'send_message' 이벤트를 서버로 보냅니다.

   1     function sendMessage(myMessage) {
   2       socket.emit('send_message', {
   3         room: '1', // 메인 페이지 채팅방 ID
   4         message: myMessage
   5       });
   6     }

●Swagger와 웹소켓(WebSocket)
안타깝게도, Swagger UI에서는 웹소켓(socket.io) 기능을 직접 테스트할 수 없습니다.
Swagger는 일반적인 HTTP API(GET, POST 등)를 테스트하기 위한 도구입니다. 웹소켓은 서버와 계속 연결을 유지하는 방식이라, Swagger에서 'Execute' 버튼을 한 번 누르는 것만으로는 테스트할 수 없습니다.
따라서, 채팅 기능은 프론트엔드 개발자가 실제 코드를 작성해서 테스트해야 합니다.

***추가로 저희가 메인페이지는 tn_article테이블의 topic_id=1, 정치페이지는 topic_id=2, 경제는 3, 사회는 4, 문화는 5로 지정해서 [실시간 채팅방]을 구별하려고 합니다.(각 토픽에 대한 토론방도 지정해야되기 때문) 자세한 설명 원하시면 말해주세여.
